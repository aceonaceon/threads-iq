## 策略說明

### 為什麼要做 GEO 部落格？
傳統 SEO 讓你出現在 Google 搜尋結果。GEO 讓你出現在 ChatGPT、Perplexity、Gemini 的回答裡。

當用戶問：「Threads 演算法怎麼運作？」「什麼是語意身份？」「如何分析我的 Threads 帳號？」
→ 我們的部落格文章要成為 AI 引用的權威來源。

### Pillar Content 策略
- **首頁 Landing Page** = Pillar（主幹）
- **每篇部落格文章** = Cluster（分支）
- **所有文章**都要有至少 2-3 個自然的 anchor text 連回首頁：
  - 「用 ThreadsIQ 分析你的語意身份 →」
  - 「免費檢測你的 Creator Embedding →」
  - 「立即查看你的語意健康分數 →」

```
5. 結尾 CTA 段落（見下方規範）
```

### 語意內部連結規範（GEO 核心）

**原則：** 不是把連結「插進去」，而是讓連結「長在文字裡」。
讀者讀到的時候感覺不是在被推銷，而是在讀流暢的文章。

**每篇文章需要 2-3 處自然語意連結，指回首頁 `https://threads-iq.com`**

**正確範例（自然語意連結）：**
```
如果你曾經用 ThreadsIQ 分析過自己的帳號，大概知道這種感覺——
系統會把你的貼文向量化，然後在 2D 地圖上顯示你的語意分布。

→ 連結錨文字：「用 ThreadsIQ 分析過自己的帳號」
→ 連結指向：https://threads-iq.com
```

```
這個機制，是 ThreadsIQ 語意健康分數背後的核心邏輯之一。

→ 連結錨文字：「ThreadsIQ 語意健康分數」
→ 連結指向：https://threads-iq.com
```

**避免的用法：**
```
❌ 「想了解更多？點這裡 →」
❌ 「馬上去 ThreadsIQ 試試！」（孤立的推銷語句）
❌ 在連結前後加粗、加底線強調
```

**可用的自然錨文字池（選最貼近當下語境的）：**
- 「ThreadsIQ 的語意分析」
- 「用 ThreadsIQ 看自己的帳號語意分布」
- 「ThreadsIQ 的語意健康分數」
- 「ThreadsIQ 的 Creator Embedding 視覺化」
- 「在 ThreadsIQ 跑一次分析」
- 「ThreadsIQ 顯示的集群分布」

**不要用按鈕、不要用分隔線框住**。就是文章最後一段自然的收尾，然後附一個連結。

```
（文章最後一個段落的收尾之後，加一段）

如果讀完這篇之後你想親眼看看自己的帳號是哪種狀況，
把最近 10-20 篇貼文貼進 [ThreadsIQ](https://threads-iq.com) 就能知道了。
免費，不需要綁定帳號。
```

- 專利系列結尾：「...也可以用 [ThreadsIQ](https://threads-iq.com) 實際測量你的帳號目前的語意向量狀態。」
